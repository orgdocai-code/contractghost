# ContractGhost Channel Pivot — 2026-03-15

## Why this exists
The main loop is stuck waiting for external conversation capture from channels that are not directly executable from this runtime.
That makes "do outreach next" too vague and too dependent on future manual action.

The anti-loop fix is to create acquisition assets that can capture intent on owned surfaces right now.

## Decision
Do not spend the next runs polishing the landing page.
Do not pretend unexecuted outreach is progress.
Instead, build pain-specific acquisition pages that:
- match real freelancer search intent
- speak to one pain at a time
- point to the existing survey / waitlist flow
- can be shared manually later without extra prep
- create a base for SEO or direct linking

## Immediate page set
1. `scope-creep-contract-checklist.html`
   - Search intent: scope creep contract wording, unlimited revisions, SOW problems
2. `late-payment-contract-red-flags.html`
   - Search intent: freelancer payment terms, late payment clauses, deposits and milestones
3. `ip-transfer-before-payment.html`
   - Search intent: client owns work before payment, copyright transfer, proposal misuse

## Success criteria
Any of these pages should be able to function as:
- a direct send link in DMs
- a search entry page later
- a clearer pain-specific first touch than the generic homepage

## Message discipline
Generic pitch: weak.
Pain-specific page + pain-specific question: stronger.

## Next move
Build the first three pages and wire each one to:
- `/#survey`
