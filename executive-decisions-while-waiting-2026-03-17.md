# ContractGhost — Executive Decisions While Waiting (2026-03-17)

## Context
Spazo-controlled posting/reply actions are still the primary blocker, but founder-process says waiting timeouts do **not** justify idle looping. This file makes the decisions that can be made unilaterally so the project can keep moving without pretending more internal collateral is the answer.

## Executive Decisions

### 1. Canonical launch path
Use this exact path for human-operated distribution until real signal says otherwise:
- `operator-index.html` → operator overview
- `spazo-action-card.html` → exact first posts / reply copy
- `proof-card.html` → default lightweight proof asset
- `three-clause-review.html` → send when trust/privacy friction appears
- `sample-report.html` → send when someone asks what they actually get
- `beta-intake-log.md` + `manual-review-request-tracker.md` → mandatory evidence capture

**Decision:** Treat this as the canonical execution stack. No new launch-routing docs unless external evidence creates a missing branch.

### 2. Offer positioning
**Primary offer:** founder-led contract risk review for freelancers.

**Not positioning as:**
- fully autonomous legal AI
- enterprise contract lifecycle management
- team workflow software
- generic legal-tech for everyone

**Decision:** Narrow to freelancers/independents with immediate contract anxiety. Keep the value prop anchored on catching risky clauses before they sign.

### 3. Trust ladder
Default trust progression:
1. Proof card
2. 3-clause review path
3. Sample report
4. Full concierge/manual review

**Decision:** Do not ask cold prospects for a full contract first. Earn that step.

### 4. Signal threshold for keeping vs killing
ContractGhost is kept alive until one of these happens:
- **Keep / sharpen:** at least 1 real positive response, request, or serious objection with enough detail to improve the offer
- **Re-examine:** 10 real targeted touches with no replies after actual posting, not hypothetical posting
- **Kill:** repeated real distribution attempts produce clear indifference and no strong pain signal

**Decision:** Internal edits are no longer valid evidence. Only external responses count.

### 5. Scope freeze
Allowed work while waiting:
- logging responses
- tightening one broken branch exposed by a real prospect question
- end-to-end testing once deployment/unblock happens
- delivery readiness for manual reviews

Not allowed while waiting:
- another homepage rewrite
- another broad outreach strategy file
- another target list without live outreach attached
- another generic trust/pricing/copy brainstorm

**Decision:** Scope is frozen unless real-world feedback breaks it.

## What This Changes
- Makes the anti-loop rule operational instead of rhetorical
- Prevents more internal launch-doc sprawl
- Clarifies the exact stack Spazo should use when a distribution window opens
- Sets the evidence bar for the next founder-loop decisions

## Next Trigger
The next meaningful founder move should come from one of:
- a real reply
- a real manual review request
- a deployment unblock
- a clear 10-touch no-response outcome
