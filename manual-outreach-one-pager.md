# ContractGhost Manual Outreach One-Pager

## Context
Created on 2026-03-14 during founder-loop run 26. This is the shortest possible operating page for the first real validation conversation while hosting is blocked.

## What to send first
"I'm testing a tool that checks freelancer contracts before you sign and flags risky terms like vague scope, bad payment language, IP grabs, unlimited revisions, and one-sided termination. If useful, I can send a quick screenshot demo. What contract clause has burned you the hardest?"

## If they reply with pain
1. Ask how they review contracts today
2. Ask whether they caught the problem before or after signing
3. Offer the screenshot demo or text-only demo
4. Ask whether they would have used a fast pre-sign risk check
5. Ask whether they'd pay for it if it reliably caught these issues

## If they say they use ChatGPT
"Makes sense. What still feels annoying or unreliable about doing that manually? The thing I'm testing is less 'generic AI answer' and more freelancer-specific red-flag detection plus suggested pushback language."

## If they don't trust AI on contracts
"Totally fair. The goal isn't to replace legal advice. It's to catch obvious red flags early and make negotiation prep faster before you sign. Happy to show the exact output format rather than a vague pitch."

## What to log
Immediately add to `contractghost/beta-intake-log.md`:
- source
- persona
- pain story
- workaround
- would use
- would pay
- main objection
- next step

## Hard rule
A real response is now worth more than any new prep asset. No more planning docs after this unless they directly support a response already in progress.
